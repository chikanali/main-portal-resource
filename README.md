# main-portal-resource

本仓库用于归档和发布多个独立的 HTML 页面资源。这些页面不绑定特定域名或网站，主要作为静态资源的存储与分发使用。

## 项目简介

`main-portal-resource` 是一个轻量级的 HTML 页面归档仓库。仓库内的页面彼此独立，可用于展示、测试、存档或作为其他项目的引用资源。所有 HTML 文件均保持原始结构与样式，便于直接访问或嵌入使用。

## 目录说明

```
.
├── index.html          # 仓库入口页面（可选）
├── pages/              # 存放主要 HTML 页面
│   ├── example1.html
│   └── example2.html
├── assets/             # 公共资源（如样式、脚本、图片）
│   ├── css/
│   ├── js/
│   └── images/
└── README.md           # 本文件
```

- `pages/`：存放各个独立 HTML 页面。
- `assets/`：存放页面共享或引用的静态资源文件。
- 根目录下的 `index.html` 可作为导航或说明页（非必须）。

## 页面归档说明

- 每个 HTML 页面均为独立单元，不依赖仓库内其他页面。
- 页面资源（CSS、JS、图片等）建议存放于 `assets/` 目录中，便于统一管理。
- 若页面引用了外部资源（如 CDN），请在页面内注明来源。
- 页面文件名建议使用英文小写，避免特殊字符。

## 使用方式

1. 直接访问 GitHub Pages 或 Raw 文件链接打开 HTML 页面。
2. 克隆仓库到本地，在浏览器中打开对应 HTML 文件。
3. 可将页面嵌入到其他项目或框架中（需注意跨域限制）。

## 维护说明

- 新增页面：在 `pages/` 目录下添加 HTML 文件，并更新 `index.html`（如有）中的导航链接。
- 修改页面：直接编辑对应 HTML 文件，注意保持资源引用路径正确。
- 删除页面：移除文件及对应的导航链接。
- 请保持目录结构清晰，避免文件堆积在根目录。
